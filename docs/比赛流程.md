### 一、啤酒大赛完整流程

#### 阶段 1：赛前筹备（创建比赛）

1. **赛事基础配置**：主办方在后台创建比赛，设置比赛名称、日期、报名截止时间、报名费金额。
2. **分类与规则配置**：预设参赛酒款**投递组别**、**基础风格**，配置厂商报名表额外字段；配置**跨界评审、专业评审、桌长**三套评分维度（总分统一 50 分）。
3. **评审与评审桌管理**：录入 / 注册评审账号（姓名、手机号、微信号、资质）；创建评审桌（A 桌 / B 桌等），为每桌分配**1 名桌长 + N 名跨界评审 + N 名专业评审**，角色与单场比赛绑定。

#### 阶段 2：厂商报名参赛

1. **账号注册登录**：厂商用手机号注册并短信登录，填写联系人姓名、手机号、微信号。
2. **提交参赛酒款**：选择目标比赛，填写酒款信息（酒名、投递组别、基础风格、ABV、酒款简介、额外字段），单厂商可提交多款酒、参加多场比赛。
3. **匿名化与支付**：提交后系统生成唯一**UUID（匿名编码）**；厂商支付报名费，支付成功后酒款状态更新为「已报名」，可下载 UUID 二维码（现场张贴）。
4. **赛前状态等待**：厂商实时查看酒款状态（待付款→已报名→已入库），主办方线下接收酒款后手动更新为「已入库」。

#### 阶段 3：现场评审（评审 + 桌长主导，匿名评审）

1. **评审登录入场**：评审用手机号 + 验证码登录，自动匹配所属比赛、评审桌及角色。

2. **扫码获取酒款信息**：扫描酒款二维码，仅显示匿名信息（UUID、组别、风格、ABV、简介），**隐藏酒名、厂商信息**，扫码异常可手动输入 UUID 兜底。

3. 独立评分 + 写评语

   ：

   - 跨界评审：按简化维度评分，评语≥50 字；
   - 专业评审：按专业维度评分，评语≥30 字；
   - 提交后可修改，**桌长汇总前开放修改权限**。

4. 桌长汇总 + 批次晋级（Flight）

   ：

   - 每批次（Flight）约十几款酒，桌长查看同桌所有评审的原始评分与评语；
   - 填写 ** 共识分（最终分）** 与综合评语，锁定最终评分；
   - 每 Flight 勾选**2-4 款晋级酒**，进入下一轮评审。

5. **多轮评审**：重复扫码→评分→桌长晋级流程，直至该组别所有 Flight 评审完毕，确定组别候选获奖酒款。

#### 阶段 4：结果评定与发布

1. **数据汇总校验**：主办方后台查看各桌晋级酒款、评分进度，导出所有原始分、最终分数据，校验数据完整性。
2. **奖项设置**：按组别设置金银铜奖项，筛选全场总冠军候选。
3. **结果发布**：发布后酒款状态更新为「结果已出」，厂商可查看评审反馈、获奖信息。
4. **厂商后续操作**：获奖厂商下载奖状、填写奖牌 / 奖状收件地址。

### 二、最终获奖与名次决出规则

核心原则：**分数仅在同评审桌、同组别内有效，不做跨桌全场排名**（不同桌评审手感、标准差异大）。

1. **桌内最终分确定**

   评审独立打分后，桌长汇总并给出**共识分（最终分）**，作为该酒款在本桌的最终评分，原始评审打分仅作参考，不计入最终排名。

2. **批次晋级筛选**

   每个 Flight 评审结束后，桌长从该批次酒款中勾选**2-4 款晋级酒**，多轮 Flight 晋级后，筛选出**同组别内所有晋级酒款**，作为组别获奖候选。

3. **组别金银铜决出**

   对同组别候选酒款，按**桌长最终共识分排序**，取前三名，依次授予**金奖、银奖、铜奖**（可支持并列获奖）。

4. **全场总冠军决出**

   从**所有组别金奖酒款**中，由主办方 / 评审团综合评定，选出 1 款授予**全场总冠军**，无全场统一名次，仅设组别名次 + 总冠军。
