# 比赛详情页设计

本文档结合需求分析报告、比赛流程、当前比赛详情页和最新业务确认，整理主办方后台“比赛详情页”的产品设计口径。目标是让后续可以按本文档直接拆需求、改页面和补接口。

比赛详情页定位为单场比赛控制台，覆盖报名配置、酒款入库、评审人员、轮次编排、现场推进、结果发布和数据导出。

## 一、外部模式参考

本系统的后续轮次更接近“Mini-BOS / BOS、多轮评审、候选池晋级、评审面板重组、排序定奖”模式，而不是单纯评分表模式。

## 二、最新业务口径

### 1. 评审轮次

| 轮次 | 操作人 | 系统操作 | 普通评委是否需要提交 |
| --- | --- | --- | --- |
| 第一轮 | 普通评委、桌长 | 普通评委逐项打分和写备注；桌长填写共识分、综合评语并选择晋级酒款 | 需要 |
| 第二轮及以后 | 桌长 | 从上一轮晋级酒款中选择并排序 | 不需要 |
| 决赛 / 总冠军轮 | 桌长或主办方指定桌长 | 从候选池中选择、排序或确认冠军 | 不需要 |

### 2. 后续轮评审方式

第二轮及以后采用“选择 + 排序”方式进行。

- 仅桌长在系统中操作。
- 普通评委不需要在系统中提交评分、备注或确认动作。
- 后续轮不使用评分表。
- 后续轮不要求填写任何打分、备注或感受。
- 后续轮桌数可以少于第一轮，允许合并、减少或重新创建评审桌。
- 后续轮每张桌仍然必须指定 1 名桌长。
- 后续轮结果用于形成组别金银铜、总冠军候选和最终奖项结果。

### 3. 厂商结果展示

厂商结果页主要展示第一轮产生的评分反馈和桌长反馈。后续轮没有评分和备注，只展示晋级轨迹、排序结果和奖项结果。

| 数据来源 | 厂商可见内容 |
| --- | --- |
| 第一轮普通评委评分 | 各维度分数、总分、备注 |
| 第一轮桌长汇总 | 共识分、综合评语、是否晋级 |
| 第二轮及以后 | 晋级状态、排序结果、奖项结果 |

## 三、核心设计原则

### 1. 评审人员池和轮次桌任务分离

后续轮酒款数量会减少，评审桌也会重组。页面设计中需要区分两类概念：

| 概念 | 含义 | 示例 |
| --- | --- | --- |
| 评审人员池 | 本场比赛可用评委和桌长候选 | 张远、李澄、王禾 |
| 基础桌次 | 第一轮默认使用的评审组织结构 | A桌、B桌、C桌、D桌 |
| 轮次桌任务 | 某一轮实际发布给评审端或桌长端的任务 | 第一轮 A桌、第二轮 2A桌、决赛桌 |
| 酒款池 | 可被当前轮次分配的酒款集合 | 已入库酒款池、第一轮晋级池、组别金奖候选池 |

第一轮可以从基础桌次生成轮次桌任务。第二轮及以后应从上一轮晋级池重新创建轮次桌任务，允许桌数减少或完全重组。

### 2. 每轮独立配置

每一轮都应独立保存以下信息：

| 配置 | 第一轮 | 第二轮及以后 |
| --- | --- | --- |
| 评审方式 | 评分 + 备注 + 桌长汇总 | 选择 + 排序 |
| 桌次 | 可继承基础桌次 | 可合并、减少、重建 |
| 操作人 | 普通评委 + 桌长 | 桌长 |
| 酒款来源 | 已入库酒款 | 上一轮晋级 / 入选酒款 |
| 输出结果 | 共识分、综合评语、晋级标记 | 排序名次、入选标记 |
| 是否使用评分表 | 是 | 否 |

### 3. 所有复杂编排都在“轮次编排”中完成

概览页只展示状态和快捷入口。参赛酒款页只做酒款管理和入库。评审人员页只做人和基础桌次。真正的“哪一轮、哪张桌、评哪些酒、由谁操作、选几款、如何排序”全部放入轮次编排。

## 四、整体信息架构

建议比赛详情页包含以下 tab：

| Tab | 主要功能 | 阶段 |
| --- | --- | --- |
| 概览 | 当前阶段、关键指标、待处理事项、下一步动作 | 全阶段 |
| 报名配置 | 投递组别、基础风格库、报名补充字段、评审可见字段 | 草稿 / 报名前 |
| 参赛酒款 | 酒款列表、付款状态、入库状态、二维码和短编号 | 报名后 / 现场前 |
| 评审人员 | 评审账号、基础桌次、桌长候选和角色分配 | 评审准备前 |
| 轮次编排 | 每轮轮次桌、酒款分配、晋级数量、后续轮排序规则 | 评审准备 / 评审中 |
| 评分表 | 第一轮普通评委评分表、桌长共识分配置 | 评审开始前 |
| 现场进度 | 按轮次查看评分进度、桌长汇总、后续轮排序状态 | 评审中 |
| 结果发布 | 奖项确认、总冠军确认、厂商反馈发布 | 结果确认 |
| 数据导出 | 完整评分数据、晋级结果、奖项结果、操作日志 | 评审后 |

当前页面已有“评审与桌次”，建议改名为“评审人员”。新增“轮次编排”作为核心工作台。

## 五、页面头部设计

页面头部用于快速识别比赛和推进当前阶段。

### 1. 头部信息

| 信息 | 示例 |
| --- | --- |
| 比赛名称 | 2026 新建精酿啤酒赛 1 |
| 批次 / 届次 | 第一批次 |
| 当前状态 | 草稿、报名中、报名截止、评审准备、评审中、结果确认、已发布 |
| 比赛日 | 2026.08.20 |
| 报名费 | 199 元 / 款 |
| 当前轮次 | 第一轮、第二轮、决赛轮 |
| 当前评审方式 | 评分制、选择排序制 |

### 2. 主操作按钮

主操作按钮随阶段变化：

| 当前状态 | 主操作 |
| --- | --- |
| 草稿 | 发布报名 |
| 报名中 | 结束报名 |
| 报名截止 | 进入评审准备 |
| 评审准备 | 创建 / 发布第一轮 |
| 第一轮评审中 | 查看现场进度 |
| 第一轮全部完成 | 从晋级池创建第二轮 |
| 后续轮已编排 | 发布给桌长 |
| 后续轮桌长已提交 | 确认排序并锁定 |
| 结果确认 | 发布结果 |
| 已发布 | 导出数据 / 归档 |

更多菜单放低频动作，如修改报名时间、导出数据、查看操作记录、归档比赛。

## 六、概览 Tab 设计

概览页展示当前比赛最关键的状态和待办。

### 1. 顶部指标卡

| 指标 | 说明 |
| --- | --- |
| 开放报名准备 | 基础信息、投递组别、基础风格库是否完成 |
| 报名窗口 | 未发布、待开始、报名中、已截止 |
| 报名酒款 | 总数、待付款、已报名、已取消 |
| 当前轮次 | 当前推进到哪一轮 |
| 当前轮次桌数 | 例如第一轮 4 桌、第二轮 2 桌 |
| 下一阶段 | 下一步应执行的赛务动作 |

### 2. 当前需要处理

当前需要处理区域只展示会阻塞当前阶段推进的问题。

| 阶段 | 典型待处理项 |
| --- | --- |
| 草稿 | 投递组别未配置、基础风格库未选择 |
| 报名中 | 报名截止时间异常、待付款酒款过多 |
| 报名截止 | 酒款未入库、评审人员未完成 |
| 评审准备 | 第一轮未编排、评分表未确认 |
| 第一轮评审中 | 某桌评分未完成、桌长未汇总、晋级数量不符合配置 |
| 创建后续轮 | 晋级池为空、未指定桌长、仍有酒款未分配 |
| 后续轮进行中 | 桌长未提交排序、排序槽位未填满 |
| 结果确认 | 奖项未确认、总冠军未确认 |

### 3. 后续赛务卡

原“后续赛务”建议改成阶段任务卡：

| 任务 | 状态 | 操作 |
| --- | --- | --- |
| 酒款入库 | 待处理 / 已完成 | 去入库 |
| 第一轮编排 | 待配置 / 已发布 / 已锁定 | 去编排 |
| 第一轮评审 | 未开始 / 进行中 / 已完成 | 看进度 |
| 第二轮创建 | 待创建 / 已创建 | 从晋级池创建 |
| 第二轮排序 | 未发布 / 桌长处理中 / 已提交 | 查看排序 |
| 决赛轮 | 待创建 / 进行中 / 已提交 | 创建决赛轮 |
| 奖项确认 | 未开始 / 待确认 / 已确认 | 去发布 |

概览页只做状态摘要和快捷跳转，复杂配置进入对应 tab 完成。

## 七、报名配置 Tab 设计

报名配置保持当前方向，重点是和评审匿名信息联动。

### 1. 投递组别

- 每场比赛独立维护。
- 作为厂商报名表下拉选项。
- 作为酒款池筛选、轮次分桌和奖项确认的基础。

### 2. 基础风格库

- 默认使用全局标准风格库。
- 报名时必填、单选、可搜索。
- 评审扫码页可见。

### 3. 报名补充字段

| 字段能力 | 说明 |
| --- | --- |
| 字段类型 | 短文本、长文本、数字、单选、多选 |
| 必填规则 | 控制厂商提交校验 |
| 排序 | 控制报名页展示顺序 |
| 评审可见 | 控制是否进入匿名扫码页 |

报名开放后，投递组别、基础风格库和评审可见字段原则上锁定。确需修改时应走管理员修正流程并记录操作日志。

## 八、参赛酒款 Tab 设计

参赛酒款页负责报名酒款管理、入库确认和现场二维码准备。

### 1. 列表字段

| 字段 | 说明 |
| --- | --- |
| 匿名编号 | 评审扫码使用的 UUID |
| 短编号 | 扫码失败时手动输入 |
| 酒名 | 后台可见，评审不可见 |
| 厂商 | 后台可见，评审不可见 |
| 投递组别 | 用于筛选和分配 |
| 基础风格 | 用于评审参考 |
| 报名状态 | 待付款、已报名、已取消、结果已出 |
| 入库状态 | 待入库、已入库 |
| 轮次轨迹 | 第一轮 A桌、第二轮 2A桌、决赛桌 |
| 当前结果 | 未评审、已晋级、已淘汰、获奖 |

### 2. 必须功能

| 功能 | 说明 |
| --- | --- |
| 状态筛选 | 按付款、入库、组别、分配状态筛选 |
| 搜索 | 按匿名编号、短编号、酒名、厂商搜索 |
| 入库确认 | 单条或批量标记已入库 |
| 取消报名 | 标记已取消，取消后不可进入评审分配 |
| 二维码下载 | 单条下载、批量导出二维码标签 |
| 短编号管理 | 展示短编号，支持现场核对 |
| 轮次轨迹查看 | 查看该酒款每轮所在桌、是否晋级、最终排序 |

第一轮酒款分配只能选择已入库酒款。后续轮酒款池只来自上一轮晋级或入选酒款。

## 九、评审人员 Tab 设计

评审人员页负责本场比赛可用评审和第一轮基础桌次。

### 1. 页面职责

| 能力 | 说明 |
| --- | --- |
| 评审池 | 展示已启用评审账号 |
| 桌长候选 | 根据资质或标记筛选可担任桌长的人 |
| 基础桌次 | 创建 A 桌、B 桌等第一轮默认桌次 |
| 角色 | 分配桌长、专业评审、跨界评审 |
| 校验 | 每张基础桌必须有且只有 1 名桌长 |

### 2. 与轮次编排的关系

评审人员页维护“人”和“第一轮基础桌次”。轮次编排页维护“每一轮实际使用的轮次桌任务”。

| 场景 | 处理方式 |
| --- | --- |
| 第一轮 | 从基础桌次生成第一轮 A/B/C/D 桌 |
| 第二轮 | 从第一轮晋级池创建 2A/2B，可重新指定桌长 |
| 决赛轮 | 创建“决赛桌”，指定 1 名桌长或主办方指定负责人 |

后续轮桌长可以来自第一轮桌长，也可以换成其他桌长候选。系统只要求每张后续轮桌绑定 1 名桌长。

## 十、轮次编排 Tab 设计

轮次编排是比赛详情页的核心新增模块。

### 1. 轮次编排总布局

推荐使用四层可视化：

| 层级 | 视觉形态 | 用途 |
| --- | --- | --- |
| 顶部轮次流 | 第一轮 -> 第二轮 -> 决赛轮 -> 结果发布 | 让用户知道当前处在哪一轮 |
| 轮次概况条 | 当前轮类型、桌数、酒款数、目标数、状态 | 快速判断本轮是否可发布 |
| 三栏工作台 | 左侧酒款池、中间桌卡、右侧校验 / 配置 | 完成实际编排 |
| 晋级路径图 | A桌 -> 2A桌 -> 决赛桌 -> 奖项 | 复盘和结果确认 |

### 2. 顶部轮次流

顶部用横向步骤条展示：

| 节点 | 显示内容 |
| --- | --- |
| 第一轮 | 评分制，A/B/C/D 桌，已发布 / 进行中 / 已锁定 |
| 第二轮 | 选择排序制，2A/2B 桌，待创建 / 已发布 / 已提交 |
| 决赛轮 | 选择排序制，决赛桌，待创建 / 已提交 |
| 结果发布 | 奖项确认，未发布 / 已发布 |

点击节点切换当前轮次。未创建的后续轮显示“从晋级池创建”。

### 3. 三栏工作台

| 区域 | 内容 | 关键交互 |
| --- | --- | --- |
| 左侧：酒款池 | 当前轮可分配酒款 | 筛选、搜索、批量选择、拖拽 |
| 中间：桌卡区 | 当前轮所有轮次桌 | 新增桌、拖入酒款、调整桌长、设置目标 |
| 右侧：配置与校验 | 当前选中桌或整轮检查 | 缺桌长、未分配酒款、目标数异常、发布按钮 |

左侧酒款池字段：

| 字段 | 说明 |
| --- | --- |
| 匿名编号 | 评审识别 |
| 短编号 | 现场兜底 |
| 投递组别 | 筛选和奖项归属 |
| 基础风格 | 匿名评审参考 |
| 来源桌 | 后续轮显示上一轮来自哪张桌 |
| 来源结果 | 第一轮晋级、第二轮第 1 名等 |
| 分配状态 | 未分配、已在 2A 桌、已锁定 |

中间桌卡字段：

| 字段 | 示例 |
| --- | --- |
| 桌号 | A桌、2A桌、决赛桌 |
| 评审方式 | 评分制、选择排序制 |
| 桌长 | 张远 |
| 普通评委 | 第一轮显示；后续轮隐藏或显示“无需系统操作” |
| 酒款数 | 12 款 |
| 晋级 / 排序目标 | 晋级 3 款、选前 3 |
| 状态 | 草稿、已发布、处理中、已提交、已锁定 |
| 异常 | 缺桌长、目标数超过酒款数、仍有未分配酒款 |

右侧校验面板分两种状态：

| 状态 | 显示内容 |
| --- | --- |
| 未选中桌 | 整轮发布检查、未分配酒款、桌数、目标数、可发布状态 |
| 选中某桌 | 桌长选择、目标名额、酒款列表、删除 / 移动酒款、单桌异常 |

### 4. 轮次状态

| 状态 | 说明 | 可编辑范围 |
| --- | --- | --- |
| 未创建 | 仅后续轮占位 | 可从晋级池创建 |
| 草稿 | 已创建但未发布 | 可改桌、桌长、酒款、目标 |
| 已发布 | 评审端或桌长端可见 | 原则上锁定编排，允许撤回后修改 |
| 进行中 | 已有评分、汇总或排序动作 | 不允许删除桌和酒款 |
| 已提交 | 桌长已完成本轮任务 | 等待主办方确认 |
| 已锁定 | 主办方确认 | 不可修改，需走更正流程 |

### 5. 第一轮编排

第一轮是评分制。

#### 第一轮配置项

| 配置项 | 说明 |
| --- | --- |
| 轮次名称 | 第一轮 |
| 评审方式 | 评分 + 备注 + 桌长汇总 |
| 评审桌 | 从基础桌次生成 A 桌、B 桌、C 桌 |
| 桌长 | 每桌必须 1 名 |
| 普通评委 | 专业评审、跨界评审 |
| 酒款范围 | 从已入库酒款池分配 |
| 晋级数量 | 每桌单独配置 |
| 评分表 | 使用评分表 tab 中配置的第一轮评分表 |

#### 第一轮酒款分配

支持以下操作：

- 按投递组别筛选酒款。
- 按匿名编号或短编号搜索。
- 手动勾选加入某张桌。
- 拖拽酒款到桌卡。
- 按数量随机分配。
- 按组别平均分配。
- 从桌中移除酒款。
- 清空当前轮重新编排。

#### 第一轮校验

| 校验 | 规则 |
| --- | --- |
| 入库 | 只有已入库酒款可分配 |
| 重复 | 同一轮同一酒款只能分配到一张桌 |
| 桌长 | 每张桌必须有且只有 1 名桌长 |
| 酒款数量 | 每张桌至少 1 款酒 |
| 晋级数量 | 晋级数量必须大于 0，且不能超过本桌酒款数量 |
| 评分表 | 专业评审、跨界评审、桌长配置必须完整 |

发布第一轮后，评审 H5 展示普通评委评分任务和桌长汇总任务。

### 6. 创建后续轮向导

当上一轮完成并锁定后，概览和轮次流显示“从晋级池创建下一轮”。点击后进入创建向导。

| 步骤 | 页面交互 | 说明 |
| --- | --- | --- |
| 1. 选择候选来源 | 默认选择上一轮晋级酒款池 | 显示总数和按组别统计 |
| 2. 选择桌次策略 | 合并为 1 桌、按组别建桌、平均分成 N 桌、沿用上一轮、手动编排 | 解决后续酒款变少导致桌次变化的问题 |
| 3. 指定桌长 | 每张新桌选择 1 名桌长 | 缺桌长不能下一步 |
| 4. 分配酒款 | 自动分配后允许拖拽微调 | 支持现场临时调整 |
| 5. 设置排序目标 | 选前 3、金银铜、总冠军候选 | 目标数不能超过本桌酒款数 |
| 6. 发布确认 | 显示本轮桌数、桌长、酒款数和目标 | 发布后只给桌长端生成任务 |

#### 桌次策略

| 策略 | 适用场景 | 示例 |
| --- | --- | --- |
| 合并为 1 桌 | 晋级酒款很少，现场集中评 | A/B/C/D 晋级共 8 款 -> 决赛桌 |
| 按组别建桌 | 每个组别独立出金银铜 | IPA 桌、拉格桌、世涛桌 |
| 按数量平均分成 N 桌 | 晋级酒款仍较多 | 24 款 -> 2A / 2B 各 12 款 |
| 沿用上一轮桌长 | 桌长基本不变，只减少酒款 | A/B/C/D -> 2A/2B |
| 手动编排 | 现场有特殊安排 | 指定张远处理 IPA，李澄处理拉格 |

### 7. 第二轮及以后编排

第二轮及以后是选择排序制。

#### 后续轮配置项

| 配置项 | 说明 |
| --- | --- |
| 轮次名称 | 第二轮、第三轮、决赛轮 |
| 评审方式 | 选择 + 排序 |
| 候选来源 | 上一轮晋级或入选酒款池 |
| 操作桌 | 2A、2B、3A、决赛桌 |
| 操作人 | 指定桌长 |
| 酒款范围 | 从上一轮晋级池分配 |
| 排序目标 | 例如选出前 3、金银铜、总冠军候选 |
| 普通评委任务 | 无系统提交任务 |

#### 后续轮页面提示

后续轮编排页应明确展示：

> 本轮由桌长进行选择和排序，普通评委无需在系统中提交评分或备注。

#### 后续轮酒款池

| 来源 | 说明 |
| --- | --- |
| 第二轮 | 第一轮已晋级酒款 |
| 第三轮 | 第二轮已入选或已排序酒款 |
| 总冠军轮 | 各投递组别金奖或主办方确认的冠军候选 |

#### 后续轮操作

- 从晋级池选择酒款加入本轮桌。
- 将酒款分配给指定桌长操作。
- 设置需要选出的数量。
- 设置排序槽位名称，例如第 1 名、第 2 名、第 3 名，或金奖、银奖、铜奖。
- 调整桌数，支持新增、合并、删除草稿桌。
- 移动酒款到另一张后续轮桌。
- 发布给桌长端。
- 查看桌长提交结果。
- 主办方确认并锁定。

#### 后续轮校验

| 校验 | 规则 |
| --- | --- |
| 候选来源 | 只能选择上一轮晋级或入选酒款 |
| 操作人 | 每个后续轮操作桌必须指定 1 名桌长 |
| 目标数量 | 排序目标数量必须大于 0，且不能超过本轮酒款数量 |
| 排序完整 | 桌长必须填满目标槽位后才能提交 |
| 重复 | 同一轮同一酒款只能进入一张桌；同一排序槽位只能放 1 款酒 |
| 锁定 | 锁定后桌长不可修改 |

### 8. 晋级路径图

晋级路径图用于结果确认和复盘，不承担主要编辑。

示例：

```text
BC-2026-IPA-0001
第一轮 A桌 晋级 -> 第二轮 2A桌 第2名 -> IPA 银奖
```

路径图字段：

| 字段 | 说明 |
| --- | --- |
| 匿名编号 | 酒款匿名识别 |
| 投递组别 | 奖项归属 |
| 第一轮桌 | A桌、B桌 |
| 第一轮结果 | 共识分、是否晋级 |
| 后续轮桌 | 2A、2B、决赛桌 |
| 后续轮排序 | 第 1 / 第 2 / 第 3 |
| 奖项 | 金奖、银奖、铜奖、总冠军 |

第一轮跨桌分数不应作为后续轮自动排序依据，后台可查看，但后续轮桌长端默认不展示跨桌分数，避免误导。

## 十一、桌长 H5 后续轮交互

第二轮及以后，桌长端不显示评分表、维度、备注、共识分输入框。页面只显示选择排序任务。

### 1. 桌长入口

桌长登录后，在比赛任务页看到：

| 任务 | 示例 |
| --- | --- |
| 第一轮汇总 | A桌评分汇总 |
| 第二轮排序 | 2A桌，选择前 3 |
| 决赛轮 | 决赛桌，确认总冠军 |

普通评委在后续轮不显示待办任务，可显示提示：

> 当前轮次由桌长完成系统操作，你无需提交评分或备注。

### 2. 后续轮排序页面

移动端建议用“候选区 + 排序槽位”的简化布局，兼容触屏操作。

| 区域 | 内容 |
| --- | --- |
| 顶部 | 轮次、桌号、桌长、目标，例如“选择前 3” |
| 候选区 | 当前桌所有候选酒款 |
| 排序区 | 第 1 名、第 2 名、第 3 名，或金、银、铜 |
| 淘汰区 | 可选，用于放未入选酒款 |
| 提交区 | 已选择 3 / 3，满足后可提交 |

### 3. 移动端交互

移动端拖拽不稳定，必须同时提供非拖拽操作。

| 操作 | 说明 |
| --- | --- |
| 拖拽排序 | 把候选酒款拖入排序槽位 |
| 设为名次 | 酒款卡片菜单选择“设为第 1 / 第 2 / 第 3” |
| 上移 / 下移 | 对已排序酒款调整顺序 |
| 移出排序 | 把酒款放回候选区 |
| 清空排序 | 一键重置当前排序 |
| 提交确认 | 二次确认后提交 |

### 4. 后续轮桌长端信息展示

| 信息 | 是否展示 | 说明 |
| --- | --- | --- |
| 匿名编号 | 展示 | 现场识别酒款 |
| 短编号 | 展示 | 兜底核对 |
| 投递组别 | 展示 | 判断奖项归属 |
| 基础风格 | 展示 | 评审参考 |
| 来源桌 | 可展示 | 方便现场找酒 |
| 第一轮共识分 | 默认不展示 | 避免跨桌分数被直接当作排名依据 |
| 酒名 | 不展示 | 保持盲评 |
| 厂商 | 不展示 | 保持盲评 |

### 5. 提交校验

| 校验 | 规则 |
| --- | --- |
| 排序数量 | 必须填满目标数量 |
| 重复 | 同一酒款不能占多个名次 |
| 空槽位 | 排序槽位不可为空 |
| 锁定 | 主办方锁定后不可修改 |

## 十二、评分表 Tab 设计

评分表只服务第一轮评分制。

### 1. 评分表类型

| 类型 | 用途 |
| --- | --- |
| 跨界评审评分表 | 第一轮跨界评审使用 |
| 专业评审评分表 | 第一轮专业评审使用 |
| 桌长共识分 | 第一轮桌长汇总使用 |

### 2. 后续轮处理

第二轮及以后不使用评分表。页面应避免让用户误以为后续轮还需要配置评分维度。

建议在评分表页顶部提示：

> 评分表仅用于第一轮评分制。第二轮及以后由桌长选择并排序，不需要打分和备注。

## 十三、现场进度 Tab 设计

现场进度页按轮次切换展示。

### 1. 第一轮进度

第一轮展示评分和桌长汇总进度。

| 指标 | 说明 |
| --- | --- |
| 普通评委评分进度 | 每桌每款酒已提交人数 / 应提交人数 |
| 桌长汇总进度 | 已填写共识分和综合评语的酒款数 |
| 晋级选择 | 每桌已选晋级数 / 配置晋级数 |
| 备注字数异常 | 普通评委备注字数是否满足规则 |
| 锁定状态 | 本轮是否已由主办方锁定 |

### 2. 第二轮及以后进度

后续轮只展示桌长任务状态。

| 指标 | 说明 |
| --- | --- |
| 桌长处理状态 | 未开始、处理中、已提交 |
| 已选择数量 | 已选择数量 / 目标数量 |
| 排序完成 | 是否完成排序槽位 |
| 提交时间 | 桌长最近提交时间 |
| 锁定状态 | 主办方是否确认锁定 |

后续轮不展示普通评委评分进度、备注字数和评分表完成情况。

### 3. 桌次重组可视化

现场进度页需要展示桌次从第一轮到后续轮的变化。

| 展示方式 | 说明 |
| --- | --- |
| 轮次切换条 | 第一轮 4 桌、第二轮 2 桌、决赛 1 桌 |
| 桌卡状态 | 每张后续轮桌显示桌长、候选数、目标数、提交状态 |
| 晋级路径摘要 | 显示每个后续轮桌的候选来自哪些第一轮桌 |

示例：

```text
第一轮：A桌 12款 -> 晋级3款
第一轮：B桌 12款 -> 晋级3款
第二轮：2A桌 6款，桌长张远，选前3
```

### 4. 异常处理

| 异常 | 处理入口 |
| --- | --- |
| 第一轮某桌未完成评分 | 跳转到本桌进度明细 |
| 桌长未汇总 | 提醒桌长或现场人员 |
| 晋级数量不符合配置 | 跳转桌长汇总或后台纠正入口 |
| 创建后续轮时有晋级酒未分配 | 跳转轮次编排 |
| 后续轮桌长未提交排序 | 提醒桌长 |
| 排序结果未锁定 | 主办方确认锁定 |

## 十四、结果发布 Tab 设计

结果发布页负责把评审数据转换为厂商可见结果。

### 1. 奖项来源

| 来源 | 用途 |
| --- | --- |
| 第一轮普通评委评分 | 厂商反馈 |
| 第一轮桌长共识分和综合评语 | 厂商反馈、晋级依据 |
| 第二轮选择排序 | 组别奖项候选 |
| 决赛轮选择排序 | 总冠军候选或最终总冠军 |

### 2. 结果确认区

| 区域 | 内容 |
| --- | --- |
| 组别奖项 | 每个投递组别金奖、银奖、铜奖 |
| 总冠军 | 从组别金奖或指定候选中确认 |
| 反馈预览 | 厂商可见评分、备注、桌长综合评语 |
| 晋级轨迹 | 每款获奖酒经过哪些轮次和桌 |
| 发布检查 | 是否所有奖项、反馈和冠军都已确认 |

### 3. 发布前校验

| 校验 | 规则 |
| --- | --- |
| 第一轮反馈 | 已获奖或已参评酒款应有桌长汇总 |
| 后续轮结果 | 需要发布奖项的组别必须完成排序或人工确认 |
| 奖项重复 | 同一组别同一奖项只能绑定 1 款酒 |
| 总冠军 | 若启用总冠军奖项，必须确认 1 款酒 |
| 发布状态 | 发布后厂商 Portal 才能查看结果 |

## 十五、建议数据模型

当前数据库已有 `judge_table`、`judge_assignment`、`score_record` 等基础表，但不足以表达“每轮重组桌次”和“后续轮排序”。建议新增或演进以下概念。

### 1. competition_round

表示比赛中的一个轮次。

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| competition_id | 比赛 |
| round_no | 1、2、3 |
| round_name | 第一轮、第二轮、决赛轮 |
| round_type | SCORE、RANKING |
| source_round_id | 后续轮来源 |
| status | DRAFT、PUBLISHED、IN_PROGRESS、SUBMITTED、LOCKED |
| sort_order | 展示顺序 |

### 2. round_table

表示某轮实际使用的桌任务。

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| competition_id | 比赛 |
| round_id | 轮次 |
| table_name | A桌、2A桌、决赛桌 |
| captain_assignment_id / captain_judge_id | 本轮桌长 |
| target_count | 晋级数量或排序目标数量 |
| status | DRAFT、PUBLISHED、SUBMITTED、LOCKED |

### 3. round_table_member

表示某轮某桌的参与人员。第一轮包含普通评委和桌长；后续轮可只记录桌长，也可记录现场参与但无需提交的评委。

| 字段 | 说明 |
| --- | --- |
| round_table_id | 轮次桌 |
| judge_account_id | 评审 |
| role | CAPTAIN、PROFESSIONAL、CROSS、OBSERVER |
| system_task_required | 是否需要系统提交任务 |

### 4. round_table_entry

表示某轮某桌分配了哪些酒款。

| 字段 | 说明 |
| --- | --- |
| round_id | 轮次 |
| round_table_id | 轮次桌 |
| beer_entry_id | 酒款 |
| source_round_table_id | 来源桌 |
| status | ASSIGNED、ADVANCED、ELIMINATED、RANKED |
| sort_order | 桌内展示顺序 |

### 5. round_result

表示后续轮选择排序结果。

| 字段 | 说明 |
| --- | --- |
| round_id | 轮次 |
| round_table_id | 轮次桌 |
| beer_entry_id | 酒款 |
| result_type | ADVANCE、RANK、AWARD_CANDIDATE、CHAMPION |
| rank_no | 1、2、3 |
| slot_label | 金奖、银奖、铜奖、总冠军 |
| submitted_by | 桌长 |
| submitted_time | 提交时间 |
| locked_flag | 是否锁定 |

### 6. 与 score_record 的关系

第一轮普通评委评分和桌长共识分继续使用 `score_record`。后续轮不产生评分记录，使用 `round_result` 保存选择排序结果。

## 十七、状态与权限

### 1. 配置锁定

| 阶段 | 锁定建议 |
| --- | --- |
| 报名开放后 | 投递组别、基础风格库、评审可见字段锁定 |

## 十九、推荐落地顺序

1. 调整比赛详情页 tab，新增“轮次编排”。
2. 将现有“评审与桌次”调整为“评审人员”，保留评审角色和第一轮基础桌次配置。
3. 补充轮次数据模型：`competition_round`、`round_table`、`round_table_entry`、`round_result`。
4. 实现第一轮编排：酒款池、桌卡、晋级数量、发布到评审端。
5. 实现第一轮进度：普通评委评分、桌长汇总、晋级数量校验。
6. 实现第二轮创建向导：从第一轮晋级池生成后续轮候选，并支持桌次重组。
7. 实现后续轮桌长端：候选区、排序槽位、提交确认。
8. 实现后续轮后台查看和锁定。
9. 将后续轮排序结果接入结果发布，生成组别奖项草稿。
10. 补齐导出、操作日志、二维码和短编号能力。

## 二十、最终设计口径

比赛详情页是单场比赛控制台。报名配置决定厂商如何提交酒款，评审人员决定本场有哪些评委和第一轮基础桌次，轮次编排决定每轮实际使用哪些桌、哪些酒、哪位桌长和什么评审方式，现场进度展示执行状态，结果发布完成奖项确认和厂商反馈开放。

第一轮采用评分制，普通评委需要打分和写备注，桌长负责共识分、综合评语和晋级选择。第二轮及以后采用选择排序制，仅桌长在系统中操作，普通评委无需提交评分、备注或其他确认。

后续轮次的评审桌不是第一轮桌次的简单延续，而是从上一轮晋级池重新生成的轮次桌任务。随着酒款减少，主办方可以合并桌次、减少桌数或手动重组，但每张后续轮桌必须指定 1 名桌长。后续轮结果用于奖项确认，厂商最终可见反馈主要来自第一轮评分和桌长汇总。
