# 比赛详情页设计

本文档结合需求分析报告、比赛流程和最新业务确认，整理主办方后台“比赛详情页”的产品设计口径。比赛详情页定位为单场比赛控制台，覆盖从报名配置、酒款入库、评审编排、现场推进到结果发布的完整赛务闭环。

## 一、最新业务口径

### 1. 评审轮次口径

| 轮次 | 操作人 | 系统操作 | 普通评委是否需要提交 |
| --- | --- | --- | --- |
| 第一轮 | 普通评委、桌长 | 普通评委逐项打分和写备注；桌长填写共识分、综合评语并选择晋级酒款 | 需要 |
| 第二轮及以后 | 桌长 | 从上一轮晋级酒款中选择并排序 | 不需要 |
| 决赛 / 总冠军轮 | 桌长或主办方指定桌长 | 从候选池中选择、排序或确认冠军 | 不需要 |

### 2. 后续轮评审方式

第二轮及以后采用“选择 + 排序”方式进行。

- 仅桌长在系统中操作。
- 普通评委不需要在系统中提交评分、备注或确认动作。
- 后续轮不使用评分表。
- 后续轮不要求填写任何打分、备注或感受。
- 后续轮结果主要用于形成组别金银铜、总冠军候选和最终奖项结果。

### 3. 厂商结果展示口径

厂商结果页主要展示第一轮产生的评分反馈和桌长反馈。后续轮没有评分和备注，因此只展示晋级轨迹、排序结果和奖项结果。

| 数据来源 | 厂商可见内容 |
| --- | --- |
| 第一轮普通评委评分 | 各维度分数、总分、备注 |
| 第一轮桌长汇总 | 共识分、综合评语、是否晋级 |
| 第二轮及以后 | 晋级状态、排序结果、奖项结果 |

## 二、页面定位

比赛详情页是单场比赛的主控页面，承担以下职责：

| 职责 | 说明 |
| --- | --- |
| 配置入口 | 配置报名字段、投递组别、评分表、评审人员、轮次规则 |
| 赛务编排 | 按轮次分配评审桌、桌长、酒款和晋级数量 |
| 现场推进 | 查看当前阶段、待处理事项、评审进度和异常 |
| 结果确认 | 根据第一轮反馈和后续轮排序确认奖项 |
| 数据出口 | 导出评分、反馈、晋级、奖项和操作记录 |

比赛详情页不直接承载厂商报名表填写和评审 H5 操作。厂商提交酒款在 Portal 完成，评审和桌长操作在评审 H5 完成。比赛详情页负责配置、发布、监控、确认和导出。

## 三、整体信息架构

建议比赛详情页包含以下 tab：

| Tab | 主要功能 | 阶段 |
| --- | --- | --- |
| 概览 | 当前阶段、关键指标、待处理事项、下一步动作 | 全阶段 |
| 报名配置 | 投递组别、基础风格库、报名补充字段、评审可见字段 | 草稿 / 报名前 |
| 参赛酒款 | 酒款列表、付款状态、入库状态、二维码和短编号 | 报名后 / 现场前 |
| 评审人员 | 评审账号、基础桌次、桌长和角色分配 | 评审准备前 |
| 轮次编排 | 每轮评审桌、酒款分配、晋级数量、后续轮排序规则 | 评审准备 / 评审中 |
| 评分表 | 第一轮普通评委评分表、桌长共识分配置 | 评审开始前 |
| 现场进度 | 按轮次查看评分进度、桌长汇总、后续轮排序状态 | 评审中 |
| 结果发布 | 奖项确认、总冠军确认、厂商反馈发布 | 结果确认 |
| 数据导出 | 完整评分数据、晋级结果、奖项结果、操作日志 | 评审后 |

当前页面已有“评审与桌次”，建议调整为“评审人员”。新增“轮次编排”作为核心赛务工作台，承接酒款分配、每桌晋级数量和后续轮选择排序。

## 四、页面头部设计

页面头部用于快速识别比赛和推进当前阶段。

### 1. 头部信息

| 信息 | 示例 |
| --- | --- |
| 比赛名称 | 2026 新建精酿啤酒赛 1 |
| 批次 / 届次 | 第一批次 |
| 当前状态 | 草稿、报名中、报名截止、评审准备、评审中、结果确认、已发布 |
| 比赛日 | 2026.08.20 |
| 报名费 | 199 元 / 款 |
| 当前轮次 | 第一轮、第二轮、决赛轮 |

### 2. 主操作按钮

主操作按钮随阶段变化：

| 当前状态 | 主操作 |
| --- | --- |
| 草稿 | 发布报名 |
| 报名中 | 结束报名 |
| 报名截止 | 进入评审准备 |
| 评审准备 | 发布第一轮到评审端 |
| 第一轮评审中 | 查看现场进度 |
| 第一轮全部完成 | 创建第二轮 |
| 后续轮已编排 | 发布给桌长 |
| 后续轮桌长已提交 | 确认排序并锁定 |
| 结果确认 | 发布结果 |
| 已发布 | 导出数据 / 归档 |

“更多”菜单可放低频动作，如修改报名时间、导出数据、查看操作记录、归档比赛。

## 五、概览 Tab 设计

概览页展示当前比赛最关键的状态和待办。

### 1. 顶部指标卡

| 指标 | 说明 |
| --- | --- |
| 开放报名准备 | 基础信息、投递组别、基础风格库是否完成 |
| 报名窗口 | 未发布、待开始、报名中、已截止 |
| 报名酒款 | 总数、待付款、已报名、已取消 |
| 当前轮次 | 当前推进到哪一轮 |
| 下一阶段 | 下一步应执行的赛务动作 |

### 2. 当前需要处理

当前需要处理区域只展示会阻塞当前阶段推进的问题。

| 阶段 | 典型待处理项 |
| --- | --- |
| 草稿 | 投递组别未配置、基础风格库未选择 |
| 报名中 | 报名截止时间异常、待付款酒款过多 |
| 报名截止 | 酒款未入库、评审人员未完成 |
| 评审准备 | 第一轮未编排、评分表未确认 |
| 第一轮评审中 | 某桌评分未完成、桌长未汇总、晋级数量不符合配置 |
| 后续轮 | 第二轮未创建、桌长未提交排序 |
| 结果确认 | 奖项未确认、总冠军未确认 |

### 3. 后续赛务卡

原“后续赛务”建议改成阶段任务卡：

| 任务 | 状态 | 操作 |
| --- | --- | --- |
| 酒款入库 | 待处理 / 已完成 | 去入库 |
| 第一轮编排 | 待配置 / 已发布 / 已锁定 | 去编排 |
| 第一轮评审 | 未开始 / 进行中 / 已完成 | 看进度 |
| 第二轮创建 | 待创建 / 已创建 | 从晋级池创建 |
| 第二轮排序 | 未发布 / 桌长处理中 / 已提交 | 查看排序 |
| 奖项确认 | 未开始 / 待确认 / 已确认 | 去发布 |

概览页只做状态摘要和快捷跳转，复杂配置进入对应 tab 完成。

## 六、报名配置 Tab 设计

报名配置保持当前方向，重点是和评审匿名信息联动。

### 1. 投递组别

- 每场比赛独立维护。
- 作为厂商报名表下拉选项。
- 作为后续奖项按组别确认的基础。

### 2. 基础风格库

- 默认使用全局标准风格库。
- 报名时必填、单选、可搜索。
- 评审扫码页可见。

### 3. 报名补充字段

| 字段能力 | 说明 |
| --- | --- |
| 字段类型 | 短文本、长文本、数字、单选、多选 |
| 必填规则 | 控制厂商提交校验 |
| 排序 | 控制报名页展示顺序 |
| 评审可见 | 控制是否进入匿名扫码页 |

报名开放后，投递组别、基础风格库和评审可见字段原则上锁定。确需修改时应走管理员修正流程并记录操作日志。

## 七、参赛酒款 Tab 设计

参赛酒款页负责报名酒款管理、入库确认和现场二维码准备。

### 1. 列表字段

| 字段 | 说明 |
| --- | --- |
| 匿名编号 | 评审扫码使用的 UUID |
| 短编号 | 扫码失败时手动输入 |
| 酒名 | 后台可见，评审不可见 |
| 厂商 | 后台可见，评审不可见 |
| 投递组别 | 用于筛选和分配 |
| 基础风格 | 用于评审参考 |
| 报名状态 | 待付款、已报名、已取消、结果已出 |
| 入库状态 | 待入库、已入库 |
| 轮次分配 | 已分配到哪一轮哪张桌 |

### 2. 必须功能

| 功能 | 说明 |
| --- | --- |
| 状态筛选 | 按付款、入库、组别、分配状态筛选 |
| 搜索 | 按匿名编号、短编号、酒名、厂商搜索 |
| 入库确认 | 单条或批量标记已入库 |
| 取消报名 | 标记已取消，取消后不可进入评审分配 |
| 二维码下载 | 单条下载、批量导出二维码标签 |
| 短编号管理 | 展示短编号，支持现场核对 |

第一轮酒款分配只能选择已入库酒款。

## 八、评审人员 Tab 设计

评审人员页负责本场比赛可用评审和基础桌次。

### 1. 页面职责

| 能力 | 说明 |
| --- | --- |
| 评审池 | 展示已启用评审账号 |
| 桌次 | 创建 A 桌、B 桌等基础桌次 |
| 角色 | 分配桌长、专业评审、跨界评审 |
| 校验 | 每桌必须有且只有 1 名桌长 |

### 2. 与轮次编排的关系

评审人员页维护“人和基础桌次”。轮次编排页维护“某一轮用哪些桌、哪些酒、晋级几款”。

默认情况下，第一轮可继承评审人员页的基础桌次和成员。第二轮及以后可继续沿用，也可指定新的桌长操作桌，例如 2A、2B、3A。

## 九、轮次编排 Tab 设计

轮次编排是比赛详情页的核心新增模块。

### 1. 页面布局

建议采用三栏工作台：

| 区域 | 内容 |
| --- | --- |
| 左侧轮次列表 | 第一轮、第二轮、决赛轮；显示类型和状态 |
| 中间编排区 | 当前轮的评审桌、酒款列表、晋级数量或排序目标 |
| 右侧酒款池 / 配置面板 | 可分配酒款、筛选器、当前桌配置和校验 |

### 2. 轮次状态

| 状态 | 说明 |
| --- | --- |
| 草稿 | 已创建但未发布到评审端 |
| 已发布 | 评审端或桌长端可见 |
| 进行中 | 已有评分、汇总或排序提交 |
| 已提交 | 桌长已完成本轮任务 |
| 已锁定 | 主办方确认，桌长不可再改 |

### 3. 第一轮编排

第一轮是评分制。

#### 第一轮配置项

| 配置项 | 说明 |
| --- | --- |
| 轮次名称 | 第一轮 |
| 评审方式 | 评分 + 备注 + 桌长汇总 |
| 评审桌 | A 桌、B 桌、C 桌等 |
| 桌长 | 每桌必须 1 名 |
| 普通评委 | 专业评审、跨界评审 |
| 酒款范围 | 从已入库酒款池分配 |
| 晋级数量 | 每桌单独配置 |
| 评分表 | 使用评分表 tab 中配置的第一轮评分表 |

#### 第一轮酒款分配

支持以下操作：

- 按投递组别筛选酒款。
- 按匿名编号或短编号搜索。
- 手动勾选加入某张桌。
- 拖拽酒款到桌卡。
- 按数量随机分配。
- 按组别平均分配。
- 从桌中移除酒款。
- 清空当前轮重新编排。

#### 第一轮校验

| 校验 | 规则 |
| --- | --- |
| 入库 | 只有已入库酒款可分配 |
| 重复 | 同一轮同一酒款只能分配到一张桌 |
| 桌长 | 每张桌必须有且只有 1 名桌长 |
| 酒款数量 | 每张桌至少 1 款酒 |
| 晋级数量 | 晋级数量必须大于 0，且不能超过本桌酒款数量 |
| 评分表 | 专业评审、跨界评审、桌长配置必须完整 |

发布第一轮后，评审 H5 展示普通评委评分任务和桌长汇总任务。

### 4. 第二轮及以后编排

第二轮及以后是选择排序制。

#### 后续轮配置项

| 配置项 | 说明 |
| --- | --- |
| 轮次名称 | 第二轮、第三轮、决赛轮 |
| 评审方式 | 选择 + 排序 |
| 候选来源 | 上一轮晋级酒款池 |
| 操作桌 | 2A、2B、3A 或决赛桌 |
| 操作人 | 指定桌长 |
| 酒款范围 | 从上一轮晋级池分配 |
| 排序目标 | 例如选出前 3、金银铜、总冠军候选 |
| 普通评委任务 | 无系统提交任务 |

#### 后续轮页面提示

后续轮编排页应明确展示：

> 本轮由桌长进行选择和排序，普通评委无需在系统中提交评分或备注。

#### 后续轮酒款池

| 来源 | 说明 |
| --- | --- |
| 第二轮 | 第一轮已晋级酒款 |
| 第三轮 | 第二轮已入选或已排序酒款 |
| 总冠军轮 | 各投递组别金奖或主办方确认的冠军候选 |

#### 后续轮操作

支持以下操作：

- 从晋级池选择酒款加入本轮桌。
- 将酒款分配给指定桌长操作。
- 设置需要选出的数量。
- 设置排序槽位名称，例如第 1 名、第 2 名、第 3 名，或金奖、银奖、铜奖。
- 发布给桌长端。
- 查看桌长提交结果。
- 主办方确认并锁定。

#### 后续轮校验

| 校验 | 规则 |
| --- | --- |
| 候选来源 | 只能选择上一轮晋级或入选酒款 |
| 操作人 | 每个后续轮操作桌必须指定桌长 |
| 目标数量 | 排序目标数量必须大于 0，且不能超过本轮酒款数量 |
| 排序完整 | 桌长必须填满目标槽位后才能提交 |
| 重复 | 同一排序槽位只能放 1 款酒 |
| 锁定 | 锁定后桌长不可修改 |

## 十、评分表 Tab 设计

评分表只服务第一轮评分制。

### 1. 评分表类型

| 类型 | 用途 |
| --- | --- |
| 跨界评审评分表 | 第一轮跨界评审使用 |
| 专业评审评分表 | 第一轮专业评审使用 |
| 桌长共识分 | 第一轮桌长汇总使用 |

### 2. 后续轮处理

第二轮及以后不使用评分表。页面应避免让用户误以为后续轮还需要配置评分维度。

建议在评分表页顶部提示：

> 评分表仅用于第一轮评分制。第二轮及以后由桌长选择并排序，不需要打分和备注。

## 十一、现场进度 Tab 设计

现场进度页按轮次切换展示。

### 1. 第一轮进度

第一轮展示评分和桌长汇总进度。

| 指标 | 说明 |
| --- | --- |
| 普通评委评分进度 | 每桌每款酒已提交人数 / 应提交人数 |
| 桌长汇总进度 | 已填写共识分和综合评语的酒款数 |
| 晋级选择 | 每桌已选晋级数 / 配置晋级数 |
| 备注字数异常 | 普通评委备注字数是否满足规则 |
| 锁定状态 | 本轮是否已由主办方锁定 |

### 2. 第二轮及以后进度

后续轮只展示桌长任务状态。

| 指标 | 说明 |
| --- | --- |
| 桌长处理状态 | 未开始、处理中、已提交 |
| 已选择数量 | 已选择数量 / 目标数量 |
| 排序完成 | 是否完成排序槽位 |
| 提交时间 | 桌长最近提交时间 |
| 锁定状态 | 主办方是否确认锁定 |

后续轮不展示普通评委评分进度、备注字数和评分表完成情况。

### 3. 异常处理

| 异常 | 处理入口 |
| --- | --- |
| 第一轮某桌未完成评分 | 跳转到本桌进度明细 |
| 桌长未汇总 | 提醒桌长或现场人员 |
| 晋级数量不符合配置 | 跳转桌长汇总或后台纠正入口 |
| 后续轮桌长未提交排序 | 提醒桌长 |
| 排序结果未锁定 | 主办方确认锁定 |

## 十二、结果发布 Tab 设计

结果发布页负责把评审数据转换为厂商可见结果。

### 1. 奖项来源

| 来源 | 用途 |
| --- | --- |
| 第一轮普通评委评分 | 厂商反馈 |
| 第一轮桌长共识分和综合评语 | 厂商反馈、晋级依据 |
| 第二轮选择排序 | 组别奖项候选 |
| 决赛轮选择排序 | 总冠军候选或最终总冠军 |

### 2. 结果确认区

| 区域 | 内容 |
| --- | --- |
| 组别奖项 | 每个投递组别金奖、银奖、铜奖 |
| 总冠军 | 从组别金奖或指定候选中确认 |
| 反馈预览 | 厂商可见评分、备注、桌长综合评语 |
| 发布检查 | 是否所有奖项、反馈和冠军都已确认 |

### 3. 发布前校验

| 校验 | 规则 |
| --- | --- |
| 第一轮反馈 | 已获奖或已参评酒款应有桌长汇总 |
| 后续轮结果 | 需要发布奖项的组别必须完成排序或人工确认 |
| 奖项重复 | 同一组别同一奖项只能绑定 1 款酒 |
| 总冠军 | 若启用总冠军奖项，必须确认 1 款酒 |
| 发布状态 | 发布后厂商 Portal 才能查看结果 |

## 十三、数据导出与操作记录

比赛详情页应提供数据导出和关键操作审计。

### 1. 数据导出

| 导出项 | 内容 |
| --- | --- |
| 参赛酒款 | 酒款、厂商、组别、状态、匿名编号、短编号 |
| 第一轮评分 | 普通评委原始评分、备注、提交时间 |
| 桌长汇总 | 共识分、综合评语、晋级标记 |
| 后续轮排序 | 轮次、桌长、候选酒款、排序结果 |
| 奖项结果 | 组别奖项、总冠军、发布时间 |

### 2. 操作记录

以下动作建议记录操作日志：

- 修改比赛基础信息。
- 修改报名配置。
- 修改评审桌和评审人员。

- 分配或移除酒款。
- 修改晋级数量。
- 后台更正桌长汇总或排序结果。
- 确认奖项。
- 发布结果。
- 导出敏感数据。

## 十四、状态与权限

### 1. 配置锁定

| 阶段 | 锁定建议 |
| --- | --- |
| 报名开放后 | 投递组别、基础风格库、评审可见字段锁定 |
| 第一轮发布后 | 第一轮评审桌、酒款分配、晋级数量锁定 |
| 第一轮锁定后 | 普通评委评分和桌长汇总不可修改 |

| 后续轮锁定后 | 桌长排序结果不可修改 |
| 结果发布后 | 奖项和厂商反馈不可随意修改 |

### 2. 权限边界

| 角色 | 权限 |
| --- | --- |
| 主办方管理员 | 配置比赛、编排轮次、发布轮次、确认和锁定结果 |
| 桌长 | 第一轮汇总；第二轮及以后选择排序 |
| 普通评委 | 第一轮打分和备注 |
| 厂商 | 报名、查看自己酒款状态和发布后的结果 |

## 十六、推荐落地顺序

1. 调整比赛详情页 tab，新增“轮次编排”。
2. 将现有“评审与桌次”调整为“评审人员”，保留评审角色和基础桌次配置。
3. 实现第一轮编排：酒款池、桌卡、晋级数量、发布到评审端。
4. 实现第一轮进度：普通评委评分、桌长汇总、晋级数量校验。
5. 实现第二轮创建：从第一轮晋级池生成后续轮候选。
6. 实现后续轮桌长排序：桌长端选择和排序，后台查看和锁定。
7. 将后续轮排序结果接入结果发布，生成组别奖项草稿。
8. 补齐导出、操作日志、二维码和短编号能力。

## 十七、最终设计口径

比赛详情页是单场比赛控制台。报名配置决定厂商如何提交酒款，评审人员决定谁参与比赛，轮次编排决定每轮评什么酒和怎么推进，现场进度展示执行状态，结果发布完成奖项确认和厂商反馈开放。

第一轮采用评分制，普通评委需要打分和写备注，桌长负责共识分、综合评语和晋级选择。第二轮及以后采用选择排序制，仅桌长在系统中操作，普通评委无需提交评分、备注或其他确认。后续轮结果用于奖项确认，厂商最终可见反馈主要来自第一轮评分和桌长汇总。
